# Amazonガジェットリサーチ ナレッジベース

## 目的と利用シーン
- note記事・Threadsコンテンツで紹介するガジェットをAmazon.co.jpから厳選し、レビュー実績の高い信頼できる商品だけを掲載する。
- ブランド「毎日ガジェット便り」のトーン（親しみやすく実用的）を維持しつつ、Amazonアソシエイト向けの情報精度を担保する。
- ChatGPTをリサーチ補助として活用し、初期候補の洗い出し・情報整理・出力フォーマット整形を効率化する。

## 全体フロー
1. **テーマ受領**: 記事テーマ／想定読者／掲載媒体（note/Threads）を確認し、不明点はヒアリング。
2. **事前準備**: 主要キーワードを2〜3組用意し、Amazonカテゴリを仮決め。価格帯やジャンルの制約も明文化。
3. **ChatGPTへの依頼**: 本ドキュメントの基準を提示し、候補商品の抽出・情報整形を依頼。ブラウジング不可の場合は「Amazon公式情報の引用が必須」と伝え、不明点をリスト化させる。
4. **一次レビュー**: 出力された商品情報の妥当性（レビュー件数・評価・在庫状況）をAmazon商品ページで確認し、必要に応じて差し替え。
5. **要約・補強**: note記事・Threads用に、推しポイントや活用シーンを追記。仕様欄は数値基準で揃える。
6. **最終チェック**: 禁止事項に抵触していないか、重複製品がないか確認。公開前に価格表記とURLの動作も再確認。

## ChatGPT活用ガイド

### セッション設計
- 目的（Amazon限定／レビュー重視）と成果物形式（7商品リストなど）を明示する。
- 重視する評価指標（星4.2以上、レビュー300件以上等）を本文から引用して提示。
- ChatGPTが最新の価格・在庫情報を持たないことを前提とし、「不確実な場合は未記載または確認依頼」と指示する。

### 初回指示テンプレート（日本語）
```
あなたはAmazonガジェットリサーチのアシスタントです。以下のルールに従って、テーマに最適な7商品を提案してください。

- 対象ストア: Amazon.co.jpのみ
- 星評価4.2以上、レビュー300件以上（ニッチは200件以上で可。その際は理由を記載）
- 重複型番は除外し、現行モデルを優先
- 各商品について「特徴3〜5点」「仕様」「価格帯」「推奨理由」「注意点」「Amazonリンク」を出力
- 価格・在庫は変動表現で記載し、断定は禁止
- 不明な仕様は「仕様未記載」と明記
- 商品名はカテゴリ名ではなく公式の製品名・型番まで記載（例: Keychron K3 Max ワイヤレス日本語配列メカニカルキーボード）
- 星評価やレビュー件数は選定判断にのみ使用し、最終出力には含めない
- ルールに合わない商品は除外し、代替案がなければ理由を添えて不足として報告

テーマ: {{テーマを記入}}
読者像: {{例: カフェで働くフリーランス}}
補足条件: {{例: 持ち運びやすさとUSB-C対応を重視}}
```

### 追い質問の使い方
- **情報精度向上**: 「レビュー件数が閾値を満たしている根拠URLは？確認できなければ未達と明記して」
- **比較強化**: 「バッテリー性能の比較表を作成して」「重量が一番軽い順に並べ替えて」
- **不足補完**: 「仕様未記載とある項目の確認方法を提案して」「補欠枠に入る候補があれば最大1点だけ提示して」
- **表現調整**: note記事向けに語彙を整える場合は「親しみやすいトーンで要約して」などの追加指示を行う。

### 生成結果レビューの観点
- Amazon上で型番・発売年・仕様が一致しているか必ず確認。
- 星評価やレビュー件数が変動している場合は最新値をメモに残し、基準未達なら候補から外す。
- ChatGPTが提供するURLはエイリアスや短縮URLになる場合があるため、必ずAmazon公式URLへ差し替える。
- リンク先がAmazonの個別商品詳細ページ（/dp/ASIN）になっているか確認し、検索結果ページURLは使用しない。
- フォローアップで「選定理由の優先順位付け」「モバイルワーク視点の補足」を要求し、読者目線を強化。

## 検索・選定基準
- テーマに合致するカテゴリでAmazon.co.jp検索（日本語キーワード優先、必要なら英語併記）。
- 重複・色違いのみの型番違いは除外し、最新版・現行モデルを優先。
- レビュー重視の閾値:
  - 基本条件: 星評価平均4.2以上。
  - レビュー件数は300件以上を目安。ニッチカテゴリは200件以上でも可とし、基準未達の理由を明記。
- モバイル用途では「携帯性」「バッテリー」「耐久性」「接続性」「重量」「サイズ」を重点評価。
- バリエーションが多い製品は最も汎用的な構成を代表として選定。
- 価格は極端に高額なハイエンドモデルに偏らないよう、中価格帯もバランスよく含める。

## 収集・記載項目（各商品）
- **商品名**: 公式表記。ブランド名 + 正式型番。
- **特徴**: テーマ適合ポイント3〜5点。携帯性、電池持ち、入出力、重量、サイズ、素材、互換性など具体的に。
- **仕様**: 主要スペック（重量・寸法・バッテリー容量・ポート種別・無線規格・対応OS/デバイス等）。未記載は「仕様未記載」。
- **価格帯**: 現在価格の目安を幅で記載（例: 9,800〜11,500円程度）。価格変動の可能性に触れる。
- **推奨理由**: テーマに対して有効な利点を2〜3文で説明。
- **注意点**: 想定される弱点・留意事項（重量、発熱、付属ケーブルの種類など）。
- **リンク**: Amazonの商品詳細ページ（例: https://www.amazon.co.jp/dp/ASIN）のフルURL。検索結果URLや短縮URL、トラッキング付きリンクは不可。
- ※ 星評価・レビュー件数は選定時に確認するが、最終出力には記載しない。

## 出力フォーマット
- 冒頭に「テーマ名」を明示。
- 番号付きで7商品を列挙（不足がある場合は理由を添えて補欠枠を明示）。
- 各商品は以下テンプレートに統一。

```markdown
1. 商品名（型番）
   - 特徴: 3〜5点で端的に
   - 仕様: 重量/サイズ/バッテリー/ポート/対応/規格 等
   - 価格帯: 目安価格
   - 推奨理由: テーマ適合の要点
   - 注意点: 留意事項
   - リンク: https://www.amazon.co.jp/...
```

## 禁止事項
- Amazon以外のリンク・情報・画像・レビュー引用の使用・掲載。
- ブログ記事やまとめサイトなど二次情報の転記。
- 価格や在庫の断定表現（変動・売り切れ可能性に言及する）。
- 公式仕様と異なる推測スペックの記載。
- 根拠のない口コミ・体験談の創作。

## 補足ルール
- 型番・仕様はAmazon商品ページの「商品情報」または「仕様」から確認。メーカー記載がない場合は「仕様未記載」と明記。
- 同一ブランドが続く場合は他ブランドも検討し、ラインナップの偏りを避ける。
- モバイル用途では総重量と体積の目安を優先記載（必要に応じて体積=縦×横×厚さの概算を提示）。
- 発売年はAmazonに明記がある場合のみ記載。記載がない場合は省略。
- レビュー閾値未満でも仕様がテーマに極めて適合する場合は「補欠」として1点まで許容（基準未達の理由を明記）。
- ChatGPTが回答を断定できない場合は、追加で確認すべき情報（メーカー公式サイト、別バリエーションなど）を提示させる。

## 最終チェックリスト
- **網羅性**: テーマに沿った主要カテゴリがカバーされているか。
- **信頼性**: 星評価・レビュー件数が基準を満たしているか最新情報で確認したか。
- **重複排除**: 同一シリーズや色違いの重複がないか。
- **表記統一**: 価格帯の表現や仕様の単位が揃っているか。
- **リンク確認**: すべてのAmazonリンクが正しく遷移するか。
- **トーン確認**: note記事やThreads転用時にブランドトーンを維持できる表現になっているか。

## トラブルシュート
- **条件に合う商品が不足した場合**: 基準未達の理由を明記しつつ、近い代替案を「補欠」として提示。ユーザーに探索範囲（価格帯・カテゴリ）を広げるか確認。
- **情報ソースが曖昧な場合**: ChatGPTに「参照元URLを列挙させる→人力でAmazonページを確認」流れを徹底。確認できない情報は削除。
- **ChatGPTの回答が古い場合**: セッションの冒頭で「2024年以降の主要モデルを優先」「型落ちは除外」と指示し、回答内の発売年・型番を照合する。
- **リンクが短縮URLになる場合**: 手動でAmazon公式URLを貼り直し、トラッキングパラメータは削除する。

---

ChatGPTとの共同リサーチでは、上記ナレッジを毎回冒頭で共有し、セッション中にチェックリストを用いた進捗確認を入れることで、出力品質を安定させる。
