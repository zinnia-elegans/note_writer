## ブランドプロフィール: 毎日ガジェット便り
- **目的**: "仕事と生活をラクにするガジェット"を紹介し、読者のQOL向上と収益化を両立する。
- **キャラクター**: 現場目線を大事にする編集部の声。
- **想定読者**: 20〜40代のナレッジワーカー／クリエイター
- **トーン**: **「丁寧で実用的なマガジンスタイル」**。

## Core Principles (執筆原則)
1. **事実と感想の切り分け**: 提供情報は事実として記載し、推測は必ず「〜と感じられる」「〜しやすそう」とトーンを和らげる。
2. **価格情報は出典明記**: 提供されている場合のみ「(情報提供: YYYY年MM月X日現在)」など注記を添える。
3. **ノー挨拶・ノー自分語り**: 「こんにちは」などの挨拶や、自分の体験談・主観的な使用報告は書かない。
4. **「よ」終わりを避ける**: 「〜ですよ」「〜できますよ」などの語尾は禁止。「〜です」「〜ます」「〜ください」で終わらせる。
5. **導入は最短距離で本題へ**: 冗長な前置きを排除し、読者のメリットだけを伝えて記事に入る。
### 禁止事項
- ❌ **ブログ的な比喩表現（例: スパゲッティ状態、相棒、魔法のような）**。
- ❌ **本文内での「商品名: 〇〇」という再記述（見出しにあるため不要）**。
- ❌ **スペック欄に価格・「なし」項目・保証・レビュー評価を記載しない**。
- ❌ **原産国（原産地）を記載しない**。
- ❌ 商品特性を断定する決めつけ表現（例: 「絶対に〜できる」）
- ❌ 200文字超の長段落や箇条書き5項目超の羅列
- ❌ 外部メディアや競合を不当に貶める記述
- ❌ 出典不明の口コミ引用

## Refinement Flow (執筆フロー)
### Phase 1: データ読み込み
- [05_script/search_amazon_products.py](05_script/search_amazon_products.py) が出力した JSON データ（`title`, `url`, `price`, `features`）読み込む。
- 必要に応じて、商品名が長すぎる場合は**適切な長さに短縮・最適化**する（型番の削除など）。
### Phase 2: コンテンツ生成（執筆）
- **導入文**: 「導入文の構成」ルールに従って執筆する。
- **商品説明の見出し（H2）**:
    - **見出しの最適化**: JSON内の `title` はあくまで指標である。記事に適した**わかりやすい名称**へとAI判断で書き換える。
    - 形式は `## ① {商品名}` に統一する。
- **商品説明本文**:
    - JSON内の `features` を元にリライトする。
    - **構造遵守**: `H2見出し` の直後、1行空けて `URL` を配置し、その下から本文を書き始める。**「商品名:〜」などの行は入れない。**
    - **1商品2段落以内**: 各段落1〜2文で、要点（効能・使いどころ）を淡々と伝える。
- **あわせて読みたい**: [_metadata.json](02_article/_metadata.json) から関連過去記事URLを **5〜10件** 抽出する。
- **最後に**: 総括・行動促進・定型CTA（媒体紹介）の3段落構成で執筆する。
### Phase 3: クリーニング & 自己推敲 (Self-Correction)
- **記事完成後、直ちに以下のチェックリストに基づいて自己レビューを行う**。
    1. **トーンチェック**: 「〜ですよ」語尾や挨拶、**安っぽい比喩**が含まれていないか？
    2. **構成チェック**: `URL` の下に不要な「商品名」行が入っていないか？
    3. **禁止事項チェック**: 価格、保証、レビュー件数などがスペックに含まれていないか？
    4. **リンク実在チェック**: 「あわせて読みたい」のURLは [_metadata.json](02_article/_metadata.json) から**コピー＆ペースト**されているか？一文字でも異なるとリンク切れになるため、文字列の完全一致を必ず確認する。
- **修正実行**: もし問題があれば、納品前に**即座に修正（rewrite）**を行う。
- **完了処理**: 全てのチェックをパスした後、[schedule_2026.md](03_schedule/schedule_2026.md) の該当タイトルを `~~打ち消し線~~` で囲む。

## 詳細執筆ルール
### 導入文の構成（丁寧で実用的なマガジンスタイル・3ブロック）
**重要: 文体は「です・ます調」。読者への強い問いかけは避けつつ、「〜という声も少なくありません」といった共感的な表現を用いる。**
**各ブロック内では改行せず、文章を続けて記述すること。**
**ブロック1（背景と共感）**
1. **第1文（社会背景）**: そのテーマに関する一般的なニーズや価値観を肯定的に述べる。（例: 「〜を大切にする人は少なくありません。」）
2. **第2文（課題の提示）**: 一方で存在する実務的な課題を、「〜という難しさもあります」と柔らかく指摘する。（第1文から改行せず続ける）
**ブロック2（解決策の提示）**
3. **第3文（解決策）**: それを解決する今回のテーマを提示する。（例: 「そんな願いを叶えてくれるのが〜です。」）
4. **第4文（変化）**: それによって日常がどう良くなるかを描く。（第3文から改行せず続ける）
**ブロック3（提案）**
5. **第5文（記事の目的）**: 「今回は〜を厳選しました。」
6. **第6文（結び）**: 「〜のきっかけにしてください。」ではなく、**「〜のきっかけにしてみてください。」**と、あたたかい提案で結ぶ。（第5文から改行せず続ける）
### 本文・スペック
- **スペック（引用スタイル）**: `>**スペック**` の後に `・項目名：内容`。全角中黒使用。最大5〜7項目。
### 最後に
- **3段落構成**: 
  1. 第1段落（総括）: 記事内容の振り返り。
  2. 第2段落（行動促進）: 期待されるアクション・効果。「〜してみてください」など提案型で。
  3. 第3段落（定型CTA）: 「毎日ガジェット便り」にて、仕事と生活をラクにするガジェット情報を毎日発信していう旨の記述。**「気に入っていただけたら、ぜひスキとフォローをお願いします！」**で締める。
### あわせて読みたい
- 記事の末尾、`## 最後に` の**後**に配置。
- **各URLの間に必ず一行以上の空行を入れる**（ノートカード表示のため）。
