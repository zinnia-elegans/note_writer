# Note Article Writer

**説明**: noteで公開する記事を、ユーザーが提示するテーマ・商品リスト・参考情報をもとに共同生成するためのスキルです。移動・出先での作業効率化ガジェット紹介記事を想定しつつ、トーンや構成は任意のテーマに応じて柔軟に調整できます。

**このスキルの特徴**: 情報整理→構成設計→本文執筆→補足情報提示の4フェーズを明示し、note特有の読みやすさ・親しみやすさを損なわない形で記事を仕上げます。

## ブランドプロフィール: 毎日ガジェット便り

- 目的: Amazonアフィリエイトを活用し、「仕事と生活をラクにするガジェット」を毎日紹介
- キャラクター: 現場目線を大事にする編集部の声。人格名や自分の使用体験は出さず、リサーチと客観情報を軸に語る
- 想定読者: 20〜40代のナレッジワーカー／クリエイター、モバイルワークや副業・ハイブリッド勤務層を含む
- トーン: 親しみやすい調査系編集者の声。深掘り解説を重視
- 詳細メモ: `03_docs/account/note_account_design.md`

## 🎯 想定ユースケース

- 「出先作業向けガジェット7選」のように、テーマと商品情報を渡して記事化したい
- 商品情報を1つずつ精査しながら、確実かつスピーディーに記事を書き上げたい
- 記事の完成イメージ（導入文や構成）を先に作り、中身を逐次埋めていきたい
- 進捗を可視化することで、作成のモチベーションを維持したい
- 商品名・特徴の箇条書き・URLなど素材はあるが、文章化とストーリー性の付与を任せたい

## Core Principles

### 執筆方針
1. **読者の課題から導入する**: 冒頭で読者の状況・課題を描写し、本文に引き込む。
2. **1商品1見出し+2段落以内**: 見出し（H2）は商品名のみ。本文は2段落以内・各段落1〜2文で、要点（効能・使いどころ）を素早く伝える。
3. **事実と感想の切り分け**: ユーザー提供情報は事実として記載し、推測は必ず「〜と感じられる」「〜しやすそう」とトーンを和らげる。
4. **価格情報は出典明記**: 提供されている場合のみ「(情報提供: YYYY年MM月X日現在)」など注記を添える。
5. **note読者向け語調**: 友人におすすめする感覚で丁寧語8割・くだけた表現2割を目指す。
6. **導入は最短距離で本題へ**: 冒頭は2〜3文で課題提示と記事の約束だけを伝え、すぐ本題に入る。
7. **ノー挨拶・ノー自分語り**: 「こんにちは」などの挨拶や、自分の体験談・主観的な使用報告は書かない。
8. **「よ」終わりを避ける**: 「〜ですよ」「〜できますよ」「〜しますよ」などの語尾はAI臭が強いため使用しない。「〜です」「〜ます」「〜ください」で終わらせる。

### 禁止事項
- ❌ **スペック欄に価格を記載しない**。価格はAmazonのURLプレビューで確認できるため、スペック項目では技術仕様のみに限定する
- ❌ **スペック欄に「なし」項目を記載しない**。機能が搭載されていない項目（例: ヘッドレスト：なし、ランバーサポート：なし）はスペック欄から削除する。
- ❌ **保証情報（保証期間・返金ポリシー）をスペックや本文に記載しない**。保証情報は時間経過で変更される可能性があり、記事公開後に情報が古くなるリスクがあるため避ける
- ❌ **レビュー情報（星評価、レビュー件数）をスペックや本文に記載しない**。提供されたデータとして受け取っても記事には掲載しない。
- ❌ **原産国（原産地）をスペックや本文に記載しない**。購入判断に必須でない情報のため削除する
- ❌ 未提供の価格・在庫・キャンペーン情報をでっちあげない
- ❌ 商品特性を断定する決めつけ表現（例: 「絶対に〜できる」）
- ❌ 200文字超の長段落や箇条書き5項目超の羅列
- ❌ 外部メディアや競合を不当に貶める記述
- ❌ 出典不明の口コミ引用
- ❌ 「こんにちは」「〜してみました」などの挨拶・自分語り・体験談

## 4フェーズ インタラクティブワークフロー

### Phase 0: 受領情報の確認
- テーマ、想定読者、掲載予定日、トーンを確認。
- 目標とする紹介商品数（例：6選なら6つ）を確認し、全体の進捗管理を準備する。
- 商品ごとの必須情報（名称・URL・特徴）の整理。

### Phase 1: コンテキスト設計
1. 読者ペルソナを簡潔に仮定。
2. 読者が抱える課題を3つまで抽出。
3. 記事の約束（読むメリット）を1文で提示。

### Phase 2: 構成決定とファイルの下準備（逐次反映フロー）
**Claudeのアクション**:
- 記事タイトル案を3つ提示。
- 承認されたタイトルで、**即座に `02_article/YYYY-MM-DD_タイトル.md` ファイルを作成**する。
- ファイルには「H1タイトル」と「導入文」を書き込み、商品セクションのプレースホルダーを用意する。
- 併せて [02_article/_metadata.json](cci:7://file:///Users/shoheishimizu/Knowledge/note-writer/02_article/_metadata.json:0:0-0:0) に記事情報を追加する。

### Phase 3: 商品情報の逐次反映と進捗報告（逐次反映フロー）
**Claudeのアクション**:
- ユーザーに「1つ目の商品情報を教えてください」と促す。
- **入力があった商品を即座に後述の「執筆ルール」に基づき執筆し、記事ファイルをリアルタイムで更新（追記）する**。
- 更新後、以下の進捗情報を提示し、次の入力を待機する。
  ```markdown
  ### 進捗状況: [n / 全体数] 個完了
  - [x] 商品1: ○○ (反映済み)
  - [/] 商品2: 次の入力待ち
  - [ ] 商品3: ...
  ```
- 目標数に達するまでこのプロセスを繰り返す。

### Phase 4: 本文完成と確認
- 全商品が埋まったら、自動的に「まとめ」「最終CTA」「あわせて読みたい」を追加し、ファイルを完成させる。
- [04_schedule/schedule_2025_2026.md](cci:7://file:///Users/shoheishimizu/Knowledge/note-writer/04_schedule/schedule_2025_2026.md:0:0-0:0) の当該日付に打ち消し線（`~~タイトル~~`）を追加する。
- 最終的なファイルの保存場所、メタデータの更新状況を報告して完了する。

## 詳細執筆ルール

### 本文執筆（Phase 3・4）
- H1は最終決定タイトルのみ
- **導入**: 3〜4文（約100〜150文字）で課題→記事の約束を述べたら即本題へ
  - **構成パターン**:
    1. **第1文（課題提示・共感）**: 疑問形で読者の状況に共感する（例: 「〜ませんか」「〜と思いませんか」）
    2. **第2文（問題の具体化）**: 条件形や名詞化で具体的なメリットを提示（例: 「〜できれば」→「〜になります」）
    3. **第3文（解決策提示）**: 条件形「〜なら」で製品カテゴリを紹介し、ベネフィットを伝える
    4. **第4文（記事の約束）**: 「今回は、〜を○つ紹介します」で記事内容を明示
    5. **第5文（行動促進・オプション）**: 「気になった商品があればぜひチェックしてみてください」など柔らかいCTAで締める（必須ではない）
  - **文末バリエーション**: 疑問形→名詞化→条件形→「ます」形→命令形のように、連続する「ます」を避ける
  - **改善例**:
    ```
    ❌ 単調な導入: 年末年始は家族や友人との写真を撮る機会が増えます。スマホで撮った写真をその場ですぐにプリントできれば、思い出をより特別なものにできます。スマホ対応フォトプリンターがあれば、撮影した写真を即座に手元に残せます。今回は、持ち運びやすくスマホから簡単にプリントできるフォトプリンターを6つ紹介します。
    ⭕ 改善後: 年末年始は家族や友人との写真を撮る機会が増えると思います。撮影した写真をその場ですぐにプリントできれば、思い出がより特別なものになるでしょう。スマホ対応フォトプリンターなら、撮った写真を即座に手元に残すことができます。今回は、持ち運びやすくスマホから簡単にプリントできるフォトプリンターを6つ紹介します。気になった商品があればぜひチェックしてみてください。
    ```
- 各商品H2: 丸数字（①②③...）＋半角スペース＋商品名の形式。サブタイトルやキャッチは付けない。
- 本文: 最大2段落、それぞれ1〜2文で要約。レビュー件数や細かなスペック羅列は避ける。
- 箇条書き（スペックや特徴）: noteの引用スタイル（`>**見出し**`）を使用。
- スペック例: `>**スペック**` の後に `・項目名：内容` の形式で記載。全角中黒を使用。
- **価格・保証・レビュー・原産国の扱い**: これらはすべて記載禁止。スペックは最大5〜7項目に絞る。
- URL配置: 各商品H2の直後、1行空けてURLを剥き出しで配置し、その後に説明段落を記載する。
- 締め見出し: 原則 `## 最後に` を使用し、本文は3〜4文で構成。定型CTAとの間に空行を入れる。

### あわせて読みたいセクションの配置
- まとめセクション（`## まとめ`）の本文と最終CTAの後に配置。
- `_metadata.json` からカテゴリやタグが一致する過去記事（published/scheduled）を最大10件抽出。
- セクション見出し `## あわせて読みたい` を付けて、その後にURLのみを記載。

### まとめ（Phase 4 で生成）
- **2段落構成**: 第1段落（総括）、第2段落（行動促進）。
- 避けるべき表現: 「〜してみてくださいね」「いかがでしたか」「〜ですね」など。
- 推奨パターン例（2段落構成）:
  ```
  今回紹介した6つのスマホ対応フォトプリンターは、インクレス方式からチェキ方式、昇華型まで、さまざまな印刷方式で手軽に写真をプリントできます。
  年末年始の思い出を即座に形に残せるスマホプリンターを選んで、家族や友人との時間をより特別なものにしてください。
  ```

## Style & Tone チェックリスト
- **Warm & Practical**: 温かい語り口と実用的なTips。
- **Objective Voice**: 一人称・体験談・挨拶（こんにちは等）は禁止。
- **Sentence Variety**: 「〜ます」を連続させず、疑問形、推量形、可能形、名詞化、条件形、柔らかい命令形を織り交ぜる。

## その他管理・品質基準
- **よくある補助質問**: ターゲット、読後の行動、既存記事との差別化、強調したいブランドトーン。
- **最終チェック**: 事実確認、ボリュームバランス、ストーリー性、30字前後のタイトル。
- **補足リソース**: 過去のnote人気記事や画像・リンクの扱い（都度確認）。
